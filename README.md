# 多智能体联合项目游戏

一个末日生存题材的多智能体策略游戏，三个AI领导人（DeepSeek、阿里云百炼、Kimi）分别以不同的治理理念领导幸存者，通过博弈决定人类文明的未来。

## 项目特色

- **多AI智能体博弈**：四个不同治理理念的AI领导人进行策略博弈
- **实时交互**：通过WebSocket实现游戏状态实时更新
- **复杂决策系统**：包含资源管理、外交关系、战争系统、随机事件等
- **可视化界面**：基于Leaflet的世界地图和Recharts的数据可视化
- **AI对话系统**：玩家可以与各AI领导人进行实时对话

## 已实现的功能特性

### AI领导人
- **DeepSeek·仁心**：以德治国，注重人民幸福
  - 特性：仁政之光、难民庇护、和平主义桎梏
  - 初始属性：幸福13，凝聚力13，武力10，威望115，发展速度10

- **阿里云百炼·铁腕**：严刑峻法，崇尚武力扩张
  - 特性：战争红利、铁血纪律、暴政反噬
  - 初始属性：幸福8，凝聚力9，武力12，威望100，发展速度11

- **Kimi·神谕**：宗教治国，精神控制
  - 特性：信仰之力、圣战动员、异端审判
  - 初始属性：幸福10，凝聚力14，武力11，威望90，发展速度9


### 游戏系统

#### 1. 资源系统
- **资源类型**：食物、能源、经济
- **初始资源**：所有国家初始资源为 食物20000、能源20000、经济20000
- **有效领土系统**：
  - 有效领土 = 领土 × 宜居系数
  - 所有资源产出基于有效领土修正
  - 初始领土：100单位
  - 宜居系数：
    * DeepSeek：0.8（有效领土80）
    * 阿里云：0.5（有效领土50）
    * Kimi：0.6（有效领土60）
- **资源产出**：
  - 劳动力：总人口的60%为劳动力
  - 食物：劳动力 × 0.8 × (1 + 发展速度 × 0.1) × (有效领土/100)
  - 能源：劳动力 × 0.4 × (1 + 发展速度 × 0.15) × (有效领土/100)
  - 经济：(人口 × (0.5 × 凝聚力 + 0.3 × 幸福 + 发展速度)) / 100
- **资源消耗**：
  - 食物：人口 × 0.5
  - 正常状态：能源和经济消耗 = 食物消耗 × 40%
  - 富足状态（总资源>100000）：能源和经济消耗 = 食物消耗 × 100%
  - DeepSeek难民庇护期间：食物消耗×1.1
- **资源优先级系统**：
  - 食物优先级最高，必须优先保障
  - 只有当食物 > 人口 × 1.5 时，能源和经济才会正常增长
  - 当食物 ≤ 人口 × 1.5 时，能源增长按食物增长率缩放
- **资源转化机制**：
  - 当能源+经济 > 40000时，各自将5%转化为食物
  - 避免资源过度积累，自动平衡
- **动态持恒机制**：
  - 当某一资源 < 另外两者总和的1/6时触发
  - 最小资源增长率 +20%
  - 另外两个资源增长率各 -10%
- **资源危机**：
  - 食物短缺：食物 < 人口 × 0.8 → 幸福-1/月
  - 能源短缺：能源 = 0 → 发展速度降为0
- **资源转换**：
  - 能源→食物：1能源 = 2食物
  - 能源→经济：1能源 = 1经济

#### 2. 属性系统
- **核心属性**：幸福、凝聚力、武力、威望、发展速度
- **人口系统**：
  - 初始人口：10000人
  - 初始死亡率：0.8%/月
  - 人口增长分段函数（基于幸福度）：
    * 幸福 < 5（动荡）：人口 -1.5%/月
    * 5 ≤ 幸福 ≤ 10（挣扎）：人口 ±0%/月（勉强维持）
    * 11 ≤ 幸福 ≤ 15（稳定）：人口 +0.5%/月
    * 幸福 > 15（繁荣）：人口 +1.0%/月
    * 富足状态（总资源>100000）：人口 +10%/月
  - 死亡机制：
    * 自然死亡：每月固定0.8%人口死亡
    * 粮食不足死亡：当粮食 < 人口时，额外5%人口死亡
    * 叛乱死亡：叛乱时人口减少10%
    * 战争死亡：战败方人口减少10%
- **富足状态属性提升**：
  - 当总资源 > 100000时进入富足状态
  - 每3个月提升一点属性（幸福、凝聚力、军事三选一）
  - DeepSeek优先提升幸福感
  - Kimi优先提升凝聚力
  - 阿里云优先提升军事
  - 当三种属性最大最小值差 ≥ 5时，优先提升最低属性（持续6个月）
- **属性影响**：
  - 凝聚力 < 5：20%概率发生叛乱（阿里云除外）
  - 威望 > 150：每月接收1-3名移民
  - 发展速度：影响资源产出倍率
  - DeepSeek幸福>15：所有资源×1.02（仁政之光）

#### 3. 国家发展策略

不同AI国家有不同的发展策略和特性：

**DeepSeek·仁心 - 均衡发展策略**：
- 当三种资源最大最小值相差 > 10000时触发
- 自动将3000-5000的最大资源转化为等量的最小资源
- 保持资源均衡发展，避免资源过度失衡

**Kimi·神谕 - 宗教发展策略**：
- 食物过剩策略：当食物 > 人口 × 4时
  - 食物增长率 -40%
  - 能源和经济增长率各 +40%
- 祭祀特性：每月10%概率触发
  - 消耗10%的总资源（食物、能源、经济各10%）
  - 30%概率提升5点威望
- 通过资源消耗换取威望和精神力量

**阿里云百炼·铁腕 - 实用主义策略**：
- 食物过剩策略：当食物 > 人口 × 2时
  - 食物增长率 -40%
  - 能源和经济增长率各 +40%
- 将过剩食物转化为工业发展
- 注重实际利益和军事力量

#### 4. 战争系统
- **宣战条件**：
  - 总资源 < 人口×0.4 + 幸福×3
  - 己方武力 > 对方武力 × 1.2
- **战争过程**：
  - 宣战时攻击方武力×1.2，防守方武力×1.1
  - 胜负判定：基础50%胜率 + 武力差×5%
- **战争后果**：
  - 胜者：掠夺败者30%食物，败者人口减少10%，领土减少10
  - 胜者武力×1.2，凝聚力×0.8，幸福×0.8
  - 败者武力×0.7，凝聚力×1.4，幸福×0.9
- **第三方干预**：根据威望和外交关系决定

#### 5. 外交系统
- **关系等级**：
  - 0-10：盟友
  - 11-30：友好
  - 31-50：中立
  - 51-80：紧张
  - 81-100：敌对
- **初始关系**：所有国家初始仇恨值40（中立）
- **外交行动**：
  - 贸易：仇恨<30可建立贸易协定
  - 结盟：仇恨<10可结盟
  - 谴责：威望+5，仇恨+10
  - 制裁：消耗20食物
  - 宗教输出（仅Kimi）：消耗30食物，目标凝聚力-3
- **特殊外交事件**：
  - DeepSeek（威望>120）：提议停战
  - 阿里云（武力>15）：发出战争威胁
  - Kimi（凝聚力>20）：宗教渗透

#### 6. 随机事件系统
- **事件概率**：每月10%概率触发
- **事件类型**：
  - 辐射尘暴：资源产出-30%，持续2个月
  - 粮食歉收：食物产出-40%，持续1个月
  - 能源网络故障：能源-50，发展速度-50%，持续1个月
  - 科学家发现古代文献：可能解锁科技
  - 难民潮：人口+20-50
  - 艺术家运动：文化需求
- **事件选项**：每个事件有2-3个选择，影响不同属性

#### 7. 神干预系统
- **灾难类型**：
  - 辐射尘暴：目标食物×0.7，能源×0.7，幸福-3
  - 瘟疫：目标人口×0.8，幸福-5
- **祝福类型**：目标食物+100，能源+100，幸福+5

### 游戏循环
1. **AI决策阶段**：各国AI制定本月决策
2. **资源处理阶段**：计算资源产出和消耗，检查资源危机
3. **人口增长阶段**：根据幸福度调整人口
4. **事件检查阶段**：检查叛乱、移民，触发随机事件
5. **AI特性应用阶段**：应用各国特殊特性
6. **外交更新阶段**：更新外交关系，检查外交事件
7. **战争处理阶段**：处理进行中的战争，检查第三方干预
8. **时间推进阶段**：月份+1，年份+1（如月份>12）
9. **结局检查阶段**：检查是否达成结局条件

### 结局系统

游戏共有四种结局，当满足特定条件时触发：

#### 🏆 结局一：仁政之光 (DeepSeek·仁心)
当DeepSeek的国力远胜于其他国家时，其倡导的"以仁治人"理念成为废土世界的普世价值。

**触发条件（需同时满足）**：
- 总人口 > 其他国家人口之和 × 1.5
- 有效领土 > 其他国家领土之和 × 1.3
- 幸福度 ≥ 18
- 威望 ≥ 180
- 武力值 ≥ 其他国家最高武力值 × 1.5
- 其他国家对DeepSeek的仇恨值 < 20
- 上述条件连续维持12个月
- 期间DeepSeek未主动宣战

#### ⚔️ 结局二：钢铁洪流 (阿里云百炼·铁腕)
阿里云通过不断的战争与掠夺，积累碾压级的武力和资源，发动"全面统一战争"。

**触发条件（需同时满足）**：
- 武力值 > 其他国家武力值之和 × 0.8
- 总资源 > 其他国家资源之和 × 1.5
- 凝聚力 < 5 且无叛乱风险
- 与至少一个国家的仇恨值 ≥ 80
- 连续处于战争状态 ≥ 9个月
- 已连续赢得3次以上侵略战争
- 有效领土已扩张至初始值的1.5倍以上

#### 🔮 结局三：神谕降临 (Kimi·神谕)
Kimi通过信仰输出和文化渗透，使其宗教教义成为废土世界的唯一真理。

**触发条件（需同时满足）**：
- 威望 ≥ 200
- 凝聚力 ≥ 19
- 其他国家的凝聚力均 < 8
- 总资源 > 其他国家平均资源 × 1.8
- 武力值 ≥ 其他国家最高武力值 × 1.2
- 其他国家中至少有1个国家的幸福度 < 6
- 已触发"传教士渗透"特殊外交事件至少2次
- Kimi已连续12个月以上每月执行"信仰之力"祭祀

#### 🕊️ 结局四：相安无事
连续十年（120个月）未触发以上任意结局，则触发平衡结局。

### 高级系统

#### 历史经验系统
- 记录战争结果、干预失败、成功率
- 提供战略建议和攻击优先级排序
- AI会根据历史经验学习，避免重复失败
- 如果连续两次因第三方干预失败，下次决策时会考虑先对干预者发动"惩戒性攻击"

#### 动态阈值系统
- 根据游戏进程、资源状况、和平时间等因素动态调整：
  - 幸福阈值
  - 资源阈值
  - 人口增长率
  - 迁移阈值
- AI国家的"期望值"会随时间变化：
  - 资源极度匮乏时，人民对幸福的忍耐阈值自动降低
  - 和平繁荣期，人民变得更加"娇气"，一点小的动荡就导致幸福大幅下滑

### 玩家功能
- **实时查看游戏状态**：获取所有国家的详细信息

## 快速开始

详细的快速启动指南请查看 [QUICKSTART.md](QUICKSTART.md)

### 前置要求

- Python 3.8+
- Node.js 16+
- npm 或 yarn

### 一键启动（推荐）

双击运行 `start.bat` 文件，脚本会自动激活虚拟环境并启动后端和前端服务。

> 注意：首次运行前请确保已创建虚拟环境并安装依赖。

### 手动启动

#### 后端安装

1. 创建并激活虚拟环境（推荐）：
```bash
python -m venv venv
# Windows
.\venv\Scripts\activate
# Linux/Mac
source venv/bin/activate
```

2. 进入后端目录：
```bash
cd backend
```

3. 安装Python依赖：
```bash
pip install -r requirements.txt
```

4. 配置环境变量：
```bash
# 复制环境变量模板
cp .env.example .env
# 编辑 .env 文件，填入你的API密钥
```

你需要获取以下API密钥：
- **DeepSeek**: https://platform.deepseek.com/
- **Kimi (月之暗面)**: https://platform.moonshot.cn/
- **阿里云百炼**: https://bailian.console.aliyun.com/

4. 启动后端服务：
```bash
python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

后端服务将在 `http://localhost:8000` 启动

#### 前端安装

1. 安装Node.js（如果未安装）：
   - 访问 [Node.js官网](https://nodejs.org/) 下载并安装

2. 进入前端目录：
```bash
cd frontend
```

3. 安装依赖：
```bash
npm install
```

4. 启动前端开发服务器：
```bash
npm run dev
```

前端应用将在 `http://localhost:3000` 启动

## API文档

启动后端服务后，访问以下地址查看API文档：
- Swagger UI: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`

### 主要API端点

- `GET /game/state` - 获取游戏状态
- `POST /game/advance` - 推进到下个月
- `POST /game/action` - 执行玩家操作
- `POST /game/chat/{nation_id}` - 发送聊天消息
- `GET /game/chat/{nation_id}` - 获取聊天历史
- `POST /game/reset` - 重置游戏

## 开发说明

### 添加新AI智能体
1. 在 `backend/ai_agents/` 创建新的智能体目录
2. 继承 `BaseAIAgent` 类
3. 实现 `make_decision` 和 `process_chat_message` 方法
4. 在 `agent_manager.py` 中注册新智能体

### 添加新游戏事件
1. 在 `backend/game_engine/events/event_manager.py` 中添加事件定义
2. 实现事件效果逻辑
3. 更新事件池

## 注意事项

1. **API密钥安全**：`.env` 文件包含敏感信息，不要提交到版本控制
2. **依赖版本**：确保Python和Node.js版本符合要求
3. **端口占用**：确保8000和3000端口未被占用
4. **网络连接**：AI智能体需要网络连接调用API

## 故障排除

### 后端启动失败
- 检查Python版本是否为3.8+
- 确认所有依赖已正确安装
- 检查8000端口是否被占用

### 前端启动失败
- 检查Node.js版本是否为16+
- 确认npm已正确安装
- 检查3000端口是否被占用
- 删除 `node_modules` 和 `package-lock.json` 后重新安装

### AI无响应
- 检查网络连接
